评测目标

如何系统、全面地评估现有单细胞多模态整合算法在不同数据类型、模态组合、数据规模和数据质量下的可用性、准确性和鲁棒性？

数据与设计

数据类型：scRNA-seq, scATAC-seq, CITE-seq, spatial multi-omics, single-cell multiome

样本信息：人类和小鼠样本，涵盖免疫（BMMC, PBMC）、肾脏、皮肤、大脑等组织；包含14个公开数据集构建的101个基准数据集，细胞规模从1,000到700,000不等，批次数量从2到143个。

评测设计：系统性基准测试设计。收集40种算法（共65个整合方法），分为配对（paired）、非配对对角线（unpaired diagonal）和非配对马赛克（unpaired mosaic）三大类。从可用性（usability）、准确性（accuracy，包括生物学保守性和批次去除）和鲁棒性（robustness，包括抗稀疏性和多次运行稳定性）三个维度进行统一评估。

数据与代码

数据可用性：使用了14个公开的单细胞多组学数据集（具体见原文Supplementary Table 2，如BMMC Multiome, CITE-seq, SHARE-seq等）。

代码可用性：作者开发了用于访问基准测试结果的用户友好平台以及基准测试流程（benchmark pipeline）（具体链接见原文或补充材料）。

评测维度分析

#1 Usability & Scalability (可用性与可扩展性)

主要发现：基因活性矩阵（gene activity matrix, GAM）的计算成为非配对scRNA+scATAC整合的严重计算瓶颈。Signac和Seurat的GAM计算函数在处理500,000细胞时因R语言对象长度限制或内存溢出而崩溃。GLUE因不依赖GAM而成为唯一能处理该规模数据的方法。部分深度学习模型（如MIDAS, scMoMaT）因GPU内存超限被淘汰。

最佳：GLUE, MultiVI, uniPort, Seurat v5 Bridge

最差：scAI, DCCA, MIDAS, GCN-SC

意外发现：非配对整合的瓶颈往往不在于整合算法本身，而在于前置的特征工程（如GAM计算）无法扩展到单细胞图谱级别（atlas-level）的数据规模。

混杂因素：R语言底层内存管理机制限制了基于R的GAM计算扩展性，这可能使基于Python且不依赖GAM的方法在可用性评分中获得不公平的优势。

page 2443 / Figure 4a, 4b / Supplementary Table 8

#2 Accuracy & Batch Effect Correction (准确性与批次校正)

主要发现：没有单一算法在所有数据集上称霸。在配对scRNA+ADT中，提供批次（batch）信息作为协变量对totalVI和Multigrate的性能提升具有决定性作用（如totalVI的graph iLISI从0.1提升至0.4）。图输出方法（如Seurat v4 WNN）在缺乏精细细胞注释的数据集上，其批次去除指标可能因依赖图结构局部逆辛普森指数而失真。

最佳：Seurat v4 WNN, totalVI (with batch), bindSC, SpatialGlue

最差：CiteFuse, sciPENN, scMM

意外发现：SpatialGlue在生物学保守性上优于Harmony和STAligner，但在批次去除上不如STAligner，表明空间先验（spatial priors）可能限制了跨批次的全局混合。

混杂因素：评估指标（如LISI）对离散细胞类型敏感，但对连续生物学流形（如发育轨迹）的整合质量评估可能存在偏差。

page 2440-2441 / Figure 2b, 3b, 3c / Extended Data Fig. 4c, 5b

#3 Robustness & Stability (鲁棒性与稳定性)

主要发现：降采样（downsampling）对生物学保守指标的影响大于对批次去除指标的影响。在稳定性方面，部分非深度学习方法（如SIMBA, MaxFuse）在多次运行中表现出指标波动；而固定了随机种子的深度学习模型（如SCALEX, uniPort, Multigrate）输出完全确定。scDEC和scMVP在未固定种子时方差最大。

最佳：GLUE, Seurat v4 WNN, SCALEX, uniPort

最差：scDEC, scMVP (unseeded), SIMBA

意外发现：通常认为非深度学习确定性算法更稳定，但评测发现部分基于图或最优传输的非DL方法因内部启发式步骤或并行计算顺序导致多次运行结果波动。

混杂因素：深度学习框架（如PyTorch）的GPU并行计算非确定性可能导致未固定种子时的方差，这属于工程实现问题而非算法理论缺陷。

实践指南

1. 始终检查算法是否支持并正确配置了批次（batch）协变量输入，这对深度学习生成模型（如totalVI, Multigrate）的性能至关重要。2. 对于非配对RNA-ATAC整合，若数据规模超过10万细胞，应放弃依赖传统GAM计算的方法，转向基于图神经网络或直接特征映射的方法（如GLUE）。3. 在评估整合质量时，不应仅依赖单一指标（如LISI），需结合生物学保守性和下游任务（如差异表达、轨迹推断）综合判断。

评测局限性
inferred 伪非配对数据（pseudo-unpaired datasets）的构建方式是通过人为掩盖配对数据的细胞匹配信息，这无法完全模拟真实独立实验中由于供体差异、测序平台差异带来的系统性生物学偏移和技术噪声。 — 影响: 可能导致对非配对整合算法（尤其是对角线和马赛克方法）在真实世界复杂场景下泛化能力的过高估计。
inferred 评估指标（如 LISI, ARI）主要针对离散的细胞类型聚类设计，对连续生物学过程（如发育轨迹、细胞状态渐变、RNA velocity）的跨模态对齐质量缺乏有效的量化手段。 — 影响: 可能使倾向于过度聚类或破坏连续流形结构的算法在评分中获利，而保留轨迹结构的算法被低估。
stated 空间多组学（spatial multi-omics）部分的评估仅包含3个专用工具和有限的公开数据集，且未深入评估空间分辨率下的亚细胞或配体-受体相互作用推断。 — 影响: 空间整合部分的结论代表性受限，无法全面指导复杂空间微环境下的多模态分析。

评测论文阅读汇报：问题背景、方法概览、评测结果、对比讨论与实践启示。

评测背景

评测问题背景和动机

单细胞多模态测序技术（single-cell multi-modal sequencing）的爆发使得在单细胞分辨率下同时测量DNA可及性、RNA表达和蛋白质丰度成为可能。然而，不同模态在维度、稀疏性和数据分布上存在巨大异质性（如scATAC-seq的极度稀疏与二值化）。为应对这一挑战，生物信息学界开发了超过40种整合算法。然而， prior studies 仅覆盖了少量工具和特定场景，缺乏一个在统一硬件限制、多模态组合和数据规模下全面评估算法可用性、准确性和鲁棒性的系统性基准（benchmark）。

方法概览

被评测方法的技术原理

被评测的方法主要分为几大类：基于图/锚点的方法（如 Seurat v4 WNN, bindSC），通过寻找跨模态互近邻构建联合图；深度生成模型（如 MultiVI, totalVI, scMVP），利用变分自编码器（VAE）将多模态数据映射到共享潜在空间（latent space）；基于矩阵分解的方法（如 MOFA+, Liger）；以及基于图神经网络的方法（如 GLUE, SpatialGlue），利用先验网络或空间邻接图引导跨模态对齐。此外，针对非配对数据，许多方法依赖基因活性矩阵（gene activity matrix, GAM）作为桥梁。

评测结果

按维度叙述评测结果

在可用性方面，评测揭示了GAM计算是超大规模非配对整合的致命瓶颈，GLUE因绕过此步骤成为唯一支持50万细胞的方法。在准确性方面，配对scRNA+scATAC任务中Seurat v4 WNN和scMDC领先，而配对scRNA+ADT任务中totalVI和bindSC表现最佳；值得注意的是，提供批次协变量使totalVI的批次去除指标提升了数倍。在非配对对角线和马赛克任务中，GLUE和MultiVI/Seurat v5 Bridge分别凭借其架构优势和对配对桥梁的利用占据主导。在空间多组学中，SpatialGlue证明了引入空间先验对保留生物学结构的必要性。在鲁棒性方面，降采样实验表明生物学保守指标对稀疏性更敏感，且固定随机种子的深度学习模型展现出比部分非DL方法更高的运行间稳定性。

差异分析

方法间差异的原因分析

方法间的性能差异主要源于其对模态异质性和批次效应的处理机制。基于图的方法（如Seurat）在离散细胞类型分明时表现优异，但在连续流形或批次效应极强时可能受限于局部锚点的质量。深度生成模型（如scvi-tools系列）通过概率建模和显式协变量输入，在处理复杂批次效应和模态插补时更具优势，但需要足够的计算资源和合理的参数调优。不依赖中间特征（如GAM）的端到端深度学习架构（如GLUE）在可扩展性上展现出压倒性优势。

实践启示

对实际使用者的指导意义

本研究为研究人员提供了清晰的算法选择决策树：超大规模非配对RNA-ATAC首选GLUE；配对CITE-seq且需批次校正首选totalVI（务必输入batch参数）；空间多组学首选SpatialGlue。同时，评测强调了在基准测试中区分“算法理论能力”与“工程实现瓶颈”（如R语言内存限制）的重要性。

评测局限

评测设计的局限性

该基准测试的局限在于其金标准高度依赖人工注释和伪降级数据，可能无法完全反映真实独立队列中的系统性生物学偏移。此外，现有指标对连续发育轨迹的跨模态对齐评估能力不足，且空间多组学部分的工具和数据集覆盖度受限于该领域的发展阶段，结论的泛化性需在未来持续更新。

前言 · WTD

他们要做什么

本文旨在系统评估40种单细胞多模态整合算法（涵盖DNA、RNA、蛋白质和空间组学）在配对（paired）、非配对对角线（unpaired diagonal）和非配对马赛克（unpaired mosaic）数据集上的可用性、准确性和鲁棒性（Page 2437）。评估目标是为研究人员提供针对不同数据规模、模态组合和数据质量的算法选择指南（Page 2438）。

综述 · SPL

现有文献综述

单细胞多模态测序技术快速发展，产生了配对和非配对数据，推动了40多种整合工具的开发（Page 2437）。先前的研究仅对少量工具进行了初步分析，例如评估了5种scRNA和scATAC整合工具以及4种Multiome引导的马赛克整合工具（Page 2438）。

综述 · CPL

现有文献批评

不同模态在维度和数据特征上存在巨大差异，例如scATAC-seq数据具有极高的稀疏性和二值化特征，现有工具在有效利用这些异质性时面临挑战（Page 2437）。此外，先前的基准测试覆盖范围有限，未能全面评估算法在不同数据集类型、模态、规模和质量下的综合表现（Page 2438）。

综述 · GAP

研究空白

领域内缺乏一个全面、系统的基准测试框架，无法在统一的硬件限制和评估指标下，横向比较现有大量单细胞多模态整合算法在配对、非配对及马赛克场景下的性能（Page 2437-2438）。

综述 · RAT

理论依据

随着多组学数据集规模和复杂度的增加，研究人员迫切需要客观的评估结果来针对特定数据集选择最优算法（Page 2438）。同时，新算法的开发和迭代也需要标准化的测试基准来验证其实际效用（Page 2438）。

结果 · ROF

研究结果

在配对scRNA和scATAC整合中，Seurat v4 WNN、scMDC和scMVP在准确性上表现最佳，而MultiVI和uniPort在可用性和可扩展性上最优（Page 2439-2440 / Figure 2）。
在配对scRNA和ADT整合中，totalVI、Seurat v4 WNN和bindSC综合表现最好，且提供批次信息能显著提升totalVI和Multigrate的性能（Page 2441 / Figure 3）。
在空间多组学整合中，SpatialGlue在生物学保守性指标上优于Harmony和STAligner等通用批次校正方法（Page 2443 / Figure 3g）。
在非配对对角线整合中，GLUE是唯一能处理500,000细胞规模数据集的算法，且在细胞对齐准确性和鲁棒性上全面领先（Page 2443 / Figure 4）。
在非配对马赛克整合中，MultiVI、Seurat v5 Bridge和StabMap表现最优，且利用配对细胞作为桥梁能显著提升非配对细胞的对齐效果（Page 2443-2444 / Figure 5）。
结果 · RCL

与现有文献一致

与先前认知一致，提供批次（batch）信息作为输入参数能显著提升MultiVI、Multigrate和totalVI等算法的批次校正和生物学保守性指标（Page 2440, 2441 / Extended Data Fig. 4c, Fig. 3c）。
k-NN平滑处理能有效缓解单细胞数据（尤其是scATAC-seq）在细胞水平的稀疏性问题，这与先前处理稀疏数据的策略一致（Page 2440 / Extended Data Fig. 4e）。
结果 · RTC

与现有文献相反

没有一种算法在所有数据集上均表现完美，例如Seurat v4 WNN在BMMC p10等混合批次数据集上表现优异，但在10X PBMC数据集上表现不佳，表明算法性能高度依赖于数据集的具体特征（Page 2440）。
部分非深度学习方法（如SIMBA, MaxFuse）在多次重复运行中表现出指标波动，这与通常认为非深度学习确定性算法更稳定的直觉相反，而部分深度学习模型因固定随机种子反而更稳定（Page 2443 / Figure 4g, 4h）。
结论 · WTDD

他们做了什么

作者收集了40种算法（65个方法）和14个公开数据集（101个基准数据集），建立了包含可用性、准确性和鲁棒性的统一评估工作流（Page 2438 / Figure 1）。此外，作者开发了用于访问基准测试结果的在线交互平台以及自动化基准测试流程（benchmark pipeline）（Page 2438）。

结论 · RFW

未来建议

随着空间多组学和更复杂模态的出现，未来的基准测试需要持续纳入新兴算法和模态组合（Page 2443）。
需要进一步优化算法在超大规模数据集（如百万级细胞）和极度稀疏数据下的计算效率和内存占用，例如改进基因活性矩阵（GAM）的计算方法（Page 2443）。
建议开发者在发布新算法时提供标准化的接口和固定随机种子的选项，以提高结果的可重复性和跨平台兼容性（Page 2440-2443）。

评测协议

建立统一基准工作流。所有方法在相同硬件限制（500GB内存，24小时运行时间，GPU加速）下评估。对于支持随机种子的深度学习方法，在稳定性测试中评估固定种子与不固定种子的差异。对于非配对评估，使用配对数据集隐藏匹配信息构建伪非配对数据。未向默认不支持或不要求批次信息的方法提供batch参数，以保证基础比较的公平性，但单独设计了批次参数消融实验。

Ground Truth

配对数据集的已知细胞匹配信息（cell-match information）作为非配对/马赛克整合的金标准；人工精细注释的细胞类型（curated cell annotations）作为生物学保守性的金标准。

方法学洞察

关键因素

1. 数据规模与内存墙（GAM计算瓶颈直接决定了非配对方法的可用性）。2. 批次协变量的显式输入（对VAE类模型性能影响巨大）。3. 模态异质性处理（如scATAC的二值化和高稀疏性需要特定的平滑或插补策略，如k-NN smoothing）。

非关键因素

算法是否基于“深度学习”并不直接决定其输出的稳定性。未固定种子的DL模型方差大，但固定种子后完全确定；而部分基于图或最优传输的非DL方法反而因启发式步骤存在运行间波动。

参数敏感性

Batch parameter（批次参数）对 totalVI 和 Multigrate 具有极高的敏感性。例如 totalVI 的 graph iLISI 从 0.1 跃升至 0.4，ARI 从 0.37 跃升至 0.62（Page 2441 / Figure 3c）。

计算考量

设定了 500GB 内存和 24h 运行的硬性阈值。scAI（时间超限）、DCCA/scMVAE（GPU内存超限）、MIDAS（时间与内存双超限）在此阈值下被淘汰。基于Python和PyTorch/scvi-tools的方法在GPU利用率和大规模矩阵运算上普遍优于基于R的方法。

研究建议

方法选型建议

针对您的多组学因果推断和RNA-ATAC调控关联研究，强烈推荐使用 GLUE 进行非配对数据的对角线整合，其图神经网络架构能直接输出peak-gene调控权重；针对空间转录组与单细胞参考的映射，推荐使用 SpatialGlue 结合 Squidpy 进行空间域识别；针对配对CITE-seq/Multiome数据的预处理和批次校正，使用 scvi-tools 生态的 totalVI/MultiVI，并严格配置批次协变量。

管线集成

GLUE 和 SpatialGlue 均基于 PyTorch，可无缝集成到您的 Python/Scanpy/AnnData 工作流中。totalVI/MultiVI 原生支持 MuData 和 AnnData 格式，可直接作为 scvi-tools 管道的一部分。建议构建基于 MuData 的统一数据容器，利用 scvi-tools 进行降维和批次校正，随后将潜在空间（latent representation）输入 GLUE 进行跨模态调控网络推断。

遗漏评测维度

评测未覆盖跨模态因果推断（如从ATAC到RNA的定向调控推断）的准确性评估，也未评估整合后对下游xQTL/colocalization分析的增益。此外，对连续轨迹（trajectory）和RNA velocity的跨模态一致性缺乏量化指标。
